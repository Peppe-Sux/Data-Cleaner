# 📊 Data Cleaner - Rule-Based Data Cleaning Pipeline

## Overview

**Data Cleaner** is a Python pipeline designed for cleaning and standardizing structured datasets in CSV or Excel format.

The project implements a series of deterministic rules to transform dirty, inconsistent, or incomplete data into datasets ready for analysis, automation, or integration into corporate systems.

The goal is not simple data formatting, but the creation of a reliable and consistent foundation for data processing and automation workflows.

---

## ⚙️ Key Features

The pipeline automatically applies 7 data cleaning rules:

### 1. String Normalization (Strip)
Removal of leading and trailing whitespaces in all text columns.

---

### 2. Name Standardization (Proper Case)
Conversion of names into a readable format:

```bash
mArIO rOSSi → Mario Rossi 
```

---

### 3. Date Uniformity
Automatic conversion of dates into standard format:

```bash
YYYY-MM-DD
```

---

### 4. Email Validation
Filtering rows with invalid emails using regular expressions (regex).

Rows with non-compliant emails are removed.

---

### 5. Phone Number Cleaning
Removal of non-numeric characters:

```bash
(333) 123-4567 → 3331234567
```

---

### 6. Duplicate Removal
Elimination of duplicates based on the `Email` column.

---

### 7. Missing Values Management
- `Citta` → `"N/D"`
- `Budget` → `0`

---

### 💡 What happened during this execution?
1. **Name & Phone Standardization:** Text inconsistencies like `mArIO rOSSi` and `&nbsp;&nbsp;luigi verdi&nbsp;&nbsp;` were parsed into clean *Proper Case* and stripped of extra whitespaces. Complex phone formats (`(333) 123-4567`, `333.555.4444`, `+39...`) were reduced to plain numeric strings.
2. **Date Uniformity:** Inconsistent date patterns (such as `01/02/26` or `10/05/26`) were automatically detected and transformed into standard ISO `YYYY-MM-DD` format.
3. **Deduplication:** The pipeline intercepted duplicate entries for both `mArIO rOSSi` and `giovanni` based on identical email addresses, safely discarding the redundant rows.
4. **Email Validation:** `ANNA bianchi` was completely filtered out because her email address (`anna.bianchi@hotmail`) lacked a valid top-level domain (like `.com` or `.it`), failing the regex security mask.
5. **Missing Values Imputation:** Absent records (*NaN*) for `City` were falling back to `"N/A"` (as seen in Alessandro's row), while missing financial data for `Budget` was safely initialized to `0.0`.

## 🧠 Project Logic

This project follows a **rule-based deterministic data processing** approach.

It does not use machine learning models, but relies on:

- explicit rules
- vectorized transformations with Pandas
- a sequential cleaning pipeline

This makes it suitable for contexts where the following are required:
- predictability
- repeatability
- total control over transformations
